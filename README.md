# ChemAI: Predict the Cure

Решение задачи хакатона **ChemAI: Predict the Cure**.

Цель соревнования — построить ML-пайплайн, который по числовым молекулярным дескрипторам химического соединения предсказывает три показателя биологической активности против вируса гриппа:

* `IC50` — концентрация, при которой вещество подавляет 50% активности вируса;
* `CC50` — концентрация, при которой вещество токсично для 50% клеток;
* `SI` — индекс селективности.

Качество решения оценивается на Kaggle по метрике RMSE, усреднённой по трём таргетам.

**Финальный public Kaggle score:** `293.8085 RMSE`

## Краткое описание подхода

Решение построено как воспроизводимый ML-пайплайн для табличных данных.

Основные этапы:

1. Загрузка данных и проверка структуры `train` и `test`.
2. Проверка совпадения признаков в обучающей и тестовой выборках.
3. Анализ пропусков.
4. Анализ распределений целевых переменных.
5. Проверка выбросов и длинных правых хвостов у `IC50`, `CC50` и `SI`.
6. Проверка связи между `SI`, `IC50` и `CC50`.
7. Тестирование дополнительного feature engineering.
8. Сравнение базовых моделей.
9. Подбор финальной конфигурации по кросс-валидации.
10. Обучение финального seed-ансамбля на всех обучающих данных.
11. Генерация файла с предсказаниями для Kaggle.

## Анализ таргетов

Целевые переменные имеют выраженную асимметрию и выбросы. Это важно, потому что RMSE чувствителен к большим ошибкам на крупных значениях.

Поэтому в решении отдельно анализируются:

* исходные распределения таргетов;
* распределения после преобразования `log1p`;
* влияние выбросов на качество кросс-валидации;
* устойчивость моделей на центральной части train-распределения.

Для части моделей используется преобразование таргета:

```python
y_transformed = np.log1p(y)
```

Обратное преобразование выполняется так:

```python
y_pred = np.expm1(y_pred_transformed)
```

Такой подход помогает сделать обучение более устойчивым на данных с длинными правыми хвостами.

## Работа с `SI`

В обучающей выборке `SI` практически совпадает с отношением:

```text
SI = CC50 / IC50
```

Однако по условию соревнования `SI` необходимо предсказывать как отдельную переменную. Поэтому эта связь используется только для анализа данных и интерпретации.

В финальном пайплайне:

* `SI` обучается отдельной моделью;
* признаки для модели `SI` — только молекулярные дескрипторы;
* нет шага `SI = CC50 / IC50`;
* `SI` не вычисляется из предсказанных `IC50` и `CC50`.

Это сделано для соответствия правилам соревнования.

---

## Feature engineering

Была проверена гипотеза о добавлении дополнительных групповых признаков на основе исходных молекулярных дескрипторов:

* диапазоны зарядов;
* диапазоны EState;
* отношения молекулярных масс и атомных характеристик;
* агрегаты по VSA-признакам;
* агрегаты по функциональным группам `fr_*`.

После проверки на кросс-валидации дополнительный feature engineering не был включен в финальную модель, так как он не улучшил качество.

Вероятная причина — небольшой размер обучающей выборки и высокая коррелированность новых признаков с уже существующими RDKit-дескрипторами.

В финальном решении используется более консервативный подход:

* исходные молекулярные признаки;
* заполнение пропусков медианой внутри `Pipeline`;
* удаление константных признаков через `VarianceThreshold`;
* при необходимости отбор признаков через `SelectKBest`.

## Модели

На базовом этапе сравнивались несколько моделей для табличных данных:

* `ExtraTreesRegressor`;
* `RandomForestRegressor`;
* `HistGradientBoostingRegressor`.

Финальное решение использует отдельные модели для каждого таргета. Такой подход выбран потому, что `IC50`, `CC50` и `SI` имеют разные масштабы, разные распределения и по-разному реагируют на преобразование таргета.

Для `IC50` и `CC50` дополнительно используется смесь двух режимов обучения:

* модель на `log1p(target)`;
* модель на исходном target.

Это позволяет совместить устойчивость log-модели на основной массе объектов и способность raw-модели лучше учитывать большие значения.

Для `SI` используется отдельная модель, обученная напрямую на `SI`, без вычисления через `CC50 / IC50`.

---

## Валидация

Качество оценивалось с помощью кросс-валидации на обучающей выборке. Используется метрика, соответствующая логике Kaggle:

```text
score = mean(RMSE(IC50), RMSE(CC50), RMSE(SI))
```

Дополнительно анализировалось качество на центральной части train-распределения по квантилям таргетов. Это нужно из-за того, что полный RMSE на небольшой выборке сильно зависит от редких экстремальных значений.

Такая проверка используется только для анализа устойчивости модели и не использует тестовые ответы.

---

## Финальное решение

Финальное решение состоит из нескольких шагов:

1. Для каждого таргета выбирается отдельная конфигурация модели.
2. Пропуски заполняются внутри `Pipeline`.
3. Константные признаки удаляются через `VarianceThreshold`.
4. Для части моделей используется `SelectKBest`.
5. Модели обучаются на нескольких random seed.
6. Финальное предсказание получается усреднением seed-ансамбля.
7. На выходе формируется файл `submission.csv`.

Все random seed зафиксированы в ноутбуке, чтобы результат можно было воспроизвести.
